Chat vs Cowork 핵심 차이

구분 일반 Chat Cowork
파일 접근 ❌ 직접 접근 불가 ✅ 지정 폴더 접근 가능
작업 방식 한 번에 하나씩 응답 다단계 작업 자율 처리
결과물 텍스트로 방법 설명 완성된 파일/결과물 생성
자동화 ❌ 불가능 ✅ 반복 작업 자동화
컴퓨터 제어 ❌ 불가능 ✅ 화면 직접 조작 가능

📌 쉬운 설명

Chat: "이 파일을 어떻게 정리해야 할까?" → Claude가 설명해줘요

Cowork: "이 파일 정리해줘!" → Claude가 직접 열어서 정리하고 저장해줘요 ✨

Cowork 실제 사용 예시

📁 파일 정리

명령: "Downloads 폴더를 날짜별·종류별로 정리해줘"

→ Claude가 직접 폴더를 열어 분류하고 이름 변경까지 완료!

📊 보고서 작성

명령: "이 엑셀 데이터 분석해서 Word 보고서로 만들어줘"

→ 스프레드시트를 읽고 직접 문서를 생성!

🔍 리서치 + 자동화

명령: "경쟁사 3곳을 조사해서 비교 분석표 만들어줘"

→ 웹 검색 후 결과를 직접 문서로 저장!

📅 반복 업무 자동화

명령: "매주 월요일에 지난주 뉴스 요약을 정리해줘"

→ 예약된 작업으로 자동 처리 (일반 Chat에서는 불가능!)

📱 외출 중에도 가능

상황: 출장 중에 폰으로 "이 보고서 정리해줘" 전송

→ 데스크톱에서 Claude가 작업하고 결과를 모바일로 전송!

알면 조금 더 편해지는 HTML 명령어

예시 1: 문단 만들기

<p>이것은 한 문단입니다.</p> <p>이것은 또 다른 문단입니다.</p>

✅ 글자 앞뒤로 자동 줄바꿈이 생겨요

예시 2: 제목 만들기

<h1>제일 중요한 제목</h1> <h2>그 다음 제목</h2> <h6>가장 작은 제목</h6>

✅ h1부터 h6까지 6단계 (1이 가장 크고 중요)

예시 3: 링크 만들기

<a href="https://google.com">구글 가기</a>

✅ href 안에 이동할 주소를 적으면 클릭 가능해요

VS CODE로 간단히 수정하기

예시 4: 이미지 보여주기

<img src="image.jpg" alt="설명">

✅ src는 이미지 주소, alt는 이미지가 안 보일 때의 대체 글

예시 5: 영역 나누기

<div> <p>이 부분들이 함께 그룹화됩니다</p> </div>

✅ div로 감싸면 CSS로 한 번에 스타일 적용 가능

🎯 버튼 - 가장 자주 쓰는 태그

<button>클릭하세요</button>

✅ 이것이 가장 간단하고 자주 쓰이는 태그!
